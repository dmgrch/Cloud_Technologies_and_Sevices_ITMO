## **Лабораторная работа 2. Сравнение сервисов Amazon Web Services и Microsoft Azure. Создание единой кросс-провайдерной сервисной модели.**

### Вариант 7


### **Ход работы**

1. Сопоставление данных провайдеров: Выполнено сопоставление сервисов AWS и Azure по схожим категориям:
   - **IT Tower**
   - **Service Family**
   - **Service Type**
   - **Service Sub Type**
   - **Service Usage Type**
2. Распределение по иерархии: Сервисы классифицированы по иерархии от общего к более детализированному, сохраняя логику из лабораторной работы 1.
3. Разбиения на подтипы выполнены в нотации ЛР1. Это поможет использовать одинаковый алгоритм для анализа

### **Примеры классификации**

- **IT Tower**:
  - `Cloud Services`
  - `Database`
  - `Compute`
  - `Storage`


- **Service Family**:
  - `Analytics`
  - `Application Services`
  - `Storage & Content Delivery`
  - `AI`и др.


- **Service Type**:
  - `Compute`
  - `Object Storage`
  - `Database` и др.

- **Service Sub Type**:
  - `SQL Database`
  - `BizTalk Server`
  - `Microsoft.Batch` и другие

- **Service Usage Type**:
  - `Tools`
  - `Compute Resources`
  - `Data Storage`

### **Вывод**

- Выполнено сопоставление сервисов AWS и Azure по общим категориям.
- Построена единая сервисная модель, применимая как для AWS, так и для Azure.
